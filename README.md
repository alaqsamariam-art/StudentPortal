## 🎯 الهدف من المشروع
يهدف المشروع إلى إنشاء بوابة إلكترونية تفاعلية متكاملة تساعد في تنظيم الخدمات الطلابية وتسهيل خطوط الاتصال المباشر بين الطلاب والإدارة الجامعية. يمنح النظام وسيلة تقنية عصرية وسريعة لتقديم الشكاوى ومتابعتها لحظة بلحظة، مع توفير لوحة تحكم إدارية للمسؤولين لإدارة الحسابات والرد الفوري على الاستفسارات، مما يوفر الوقت والجهد ويحسن من كفاءة العملية التنظيمية داخل الكلية.

---

## 💡 فكرة المشروع ووصف مختصر
**بوابة الطالب الذكية** هي تطبيق ويب ديناميكي متجاوب (Dynamic Responsive Web Application) يتيح للطلبة إنشاء حسابات جديدة، تسجيل الدخول، إدارة ملفاتهم الأكاديمية والشخصية، وتنزيل المساقات المتطابقة مع تخصصاتهم. كما يوفر قسماً فريداً للشكاوى يتيح للطالب إرسال استفساره مباشرة للدعم الفني مع متابعة الردود الواردة إليه أولاً بأول مصحوبة بالإشعارات التنبيهية الفورية عند حل المشكلة.

على الجانب الآخر، يمتلك النظام **لوحة إدارة مخصصة (Admin Dashboard)** تتيح للمسؤولين إدارة وتعديل حسابات الطلاب، مراجعة وتصفية الشكاوى الواردة حسب التصنيف والحالة، وتقديم الردود الرسمية لحلها.

تم بناء وتطوير المشروع بالكامل بالاعتماد على:
* **Backend:** PHP & MySQLi
* **Frontend:** HTML5, CSS3, JavaScript, Bootstrap 5 (أيقونات ومكونات مرنة)
* **Security:** تطبيق حماية متكاملة ضد ثغرات XSS وهجمات SQL Injection لضمان سرية البيانات.

## 👥 مستخدمو النظام وصلاحياتهم (User Roles)

### 1️⃣ المسؤول (Admin)
* تسجيل الدخول الآمن للوحة التحكم.
* إدارة حسابات الطلبة بالكامل (عرض، تعديل البيانات، إضافة، حذف).
* استعراض الإحصائيات العامة للنظام ومؤشرات أعداد الطلاب والشكاوى المسجلة.
* تتبع الشكاوى الواردة، تصفيتها، وتغيير حالتها (جديدة، قيد المعالجة، مغلقة).
* تقديم الردود الرسمية على الشكاوى وإرسال تنبيهات تفاعلية تلقائية لحسابات الطلاب.

### 2️⃣ الطالب (Student)
* إنشاء حساب شخصي جديد والانضمام للبوابة الأكاديمية.
* تسجيل الدخول الآمن وحفظ الجلسة.
* تعديل الملف الشخصي وتعديل الصورة الشخصية والتخصص.
* استعراض وتسجيل المساقات الدراسية المتوافقة مع التخصص الحالي للطالب.
* استخدام مساعد الشكاوى الذكي لرفع الاستفسارات ومتابعتها.
* استقبال الإشعارات والتنبيهات الفورية بمجرد قيام الإدارة بالرد على الشكوى وإغلاقها.
